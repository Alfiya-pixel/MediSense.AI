Building a Symptom-Based Disease Diagnosis Web App with Flask and Machine Learning

Junaid Khan January 2024 6 min read

In the age of technology and information, access to accurate and timely healthcare is more critical than ever. With the increasing importance of remote healthcare solutions, we built a symptom-based disease diagnosis web application — combining Flask for the backend with a machine learning classifier to help users identify potential illnesses from reported symptoms.

The Problem

People regularly experience symptoms and want quick, reliable answers about their health. Differentiating between diseases is genuinely difficult, especially when symptoms overlap — headache and fatigue could point to anything from dehydration to dengue fever. The goal was a platform where users input symptoms and receive data-driven guidance instantly.

The Solution

We built a web app where users enter a list of symptoms and our pre-trained SVM model predicts the most likely condition. Here's how it works step by step:

  1. Symptom Input: Users enter symptoms through a clean interface supporting 132 distinct symptoms. Tags and voice input make it effortless.
  2. ML Model: An SVM classifier trained on a labeled medical dataset maps symptom vectors to disease predictions.
  3. Instant Prediction: The model returns the most probable disease in real time.
  4. Comprehensive Guidance: Alongside the prediction, users receive a disease description, precautions, medications, diet advice, and workout recommendations.

Key Features

Smart Symptom Input SVM Prediction Precaution Guidance Medication Info Diet Advice Workout Tips Voice Recognition

Technology Stack

  • Backend: Python · Flask · Pandas · NumPy · Scikit-learn
  • Frontend: HTML5 · Bootstrap 5 · Vanilla JS · Web Speech API
  • Model: Support Vector Machine (SVM) serialised via Pickle
  • Data: CSV-based medical datasets for symptoms, diseases, medications, diets, and workouts

Conclusion

MediSense brings the power of machine learning and healthcare information directly to users. It's not a replacement for professional medical advice — it's a first step: empowering people with knowledge before they see a doctor. As remote healthcare solutions gain prominence, projects like this represent a meaningful step toward democratising health information worldwide.
